407 - Trapping Rain Water II

First, we have noticed that a block on the edge can not hold any water. Supposing we have some blocks which can hold water. The height of the surface depends on the shortest block in the surrounding blocks. In other words, we can traverse the blocks by increasing order of the height. When some block nearby is lower than current block, the block can be filled with water which surface is the same height of current block. If some newly visited block is higher than current block, then the block can not hold any water.

We can calculate the maximum volume of water by the following scheme: 1. Add the edge blocks to a min heap (ordered by height). 2. Perform BFS using heap. Traverse the nearby blocks which have not been visited yet. 3. If the new block is shorter, pour some water. The volume equals to surface height - block height. Then we adjust the block height to surface height. Push the block into heap. 4. If the new block is taller, push the block into heap directly.

That being said, we dynamically add new blocks into the heap and calculate the volume of water.

815 - Bus Routes

I firstly submitted a naive solution using bfs. It considers a (transfer number, position) as state. The first solution was slow, yet acceptable.

Another solution is to find a shortest path on the graph which vertexes are buses. We can first preprocess the input by sorting bus routes and find unique stations. Then use two pointers to find intersections within different buses. S and T cannot be found in the new graph. So we should create a new S and a new T connecting to the buses reaches to the original S and T.

679 - 24 Game

Simple brutal force. I solved this using DFS. Given n numbers, select 2 of them and perform different arithmetic operations. The n - 1 numbers remaining can be treated as the same. Keep finding until an answer is found. Other wise return -1.

432 - All O`one Data Structure

Use linked list to store the values of keys in order to achieve O(1) minmax query. But how to perfrom O(1) insertion/increase/decrease/deletion operation? We can use a hash map to store the linked list node pointer which contains some particular key. Then the insersion/increase/decrease/deletion are O(1).

In C++, you can use STL to get an access to these data structures.

940 - Distinct Subsequences II

Let's take a look at the subsequences of S[1~i], including blank subsequence. Let's denote the number of subsequence of S[1~i] as dp[i]. When duplicated subsequences are not considered, we can easily find that dp[i] = dp[i-1] * 2. You can either choose to add or not to add the last character to the previous subsequences. The duplicated subsequences come from the subsequeces of the same ending character. Take "abab" as an example:

  1. dp[0] - ""
  2. dp[1] - "", "a"
  3. dp[2] - "", "a", "b", "ab"
  4. dp[3] - "", "a", "b", "ab", "a", "aa", "ba", "aba"
  5. dp[4] - "", "a", "b", "ab", "aa", "ba", "aba", "b","ab", "bb", "abb", "aab", "bab", "abab"

The duplicated "a" in dp[3] comes from dp[0] + "a". The duplicated "b" and "ab" in dp[4] comes from dp[1] + "b". The transition function can be written as: dp[i] = dp[i-1] * 2 - (dp[last(S[i]) - 1]).

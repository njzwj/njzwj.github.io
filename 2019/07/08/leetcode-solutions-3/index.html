719 - Find K-th Smallest Pair Distance

Obviously, there are \(n*(n-1)/2\) pair distances in an array of length \(n\). We can binary search for the answer. First, we sort the array in ascending order denoted as \(A = a_1, a_2, \cdots, a_n\). Then we test if an arbitrary number, say \(x\) is a K-th smallest pair distance. We just simply count how may pairs have the distance smaller than \(x\). If number of pairs is larger than \(k\), then \(x\) is larger than answer.

When counting pairs, we use the second binary search. Because \(A\) is sorted, counting pairs starting from \(a_i\) and distance smaller than \(x\) is equivalent to binary searching \(a_i + x\) between \(A[i]\) and \(A[n]\). The time complexity would be \(O(n\log(A_{max}-A_{min})\log(n))\).

632 - Smallest Range

We first consider using k pointers to solve this problem. Because the target interval (range) requires being contained to at least one element in each list, we select the smallest number in k numbers, change it to the next one in the list it belongs to and update the answer. The tricky part is to maintain the data we need: The smallest and largest number in k. Min heap would be a good choise to maintain the smallest number. The largest number can be simply maintained by updating it when inserting new number into heap. When popping the smallest number out, we have to know if this is the last one in k numbers (the duplicated situation). We use a map/hash map to maintain the number of occurance of each number. The solution is quite straightforward afterwards.

699 - Falling Squares

When a square falls, it can be held by and sticked to the highest surface that touches it (ignoring the laws of physics, the center of gravity, balance, etc.). We need to dynamically maintain the height at every position and query the max height in an abitrary interval. A segment tree is used here to maintain the height. To support interval modification and query, a lazy label is added. Because the x position has a range of \([1, 10^8]\), we need to discreterize the x position first.

45 - Jump Game II

Let's denote the least number of steps to reach position i as \(dp[i]\). It's obvious that \(dp[i]\) is non-descending. When reaching position i, we can know that \(dp[i+1]\) to \(dp[i+nums[i]]\) equal to or less than \(dp[i]+1\). Now comes the tricky part. Consider we update \(dp[i+1]\) to \(dp[i+nums[i]]\) at the time we reach i, any \(dp[i]\) updated before should be fixed. That being said, we can maintain a position representing the current farest position that can be reached in 0 step, any position within should share the same step number. If we find current position is out of reach, then we add one step to current position, and update the reach-out range. The time complexity achieves \(O(n)\) now.

664 - Strange Printer

Let's denote the minimum number of operations to get some substring as \(dp[l][r]\). If \(s[i]=s[l]\), we can split it into \(dp[l][i-1]\) and \(dp[i+1][r]\). \(s[i]\) and \(s[l]\) share one operation. A top-down dp would do the job. \(dp[i][j] = min(dp[l][i-1] + dp[i+1][r])\) for all i that \(s[i]=s[l]\). The time complexity would be \(O(n^3)\).

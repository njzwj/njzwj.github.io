472 - Concatenated Words

Because the input contains no duplicated strings, so A is strictly shorter than B for every B that is concatenated by A. We first sort all strings by length. Then find if current string can be composed by some of the strings occured before. Here we use Trie to accelerate the matching process. Search along the Trie can be faster than brutal force. Be aware that empty string cannot be concatenated by any other ones and it does no effect to the concatenation process. Thus empty string can be ignored.

446 - Arithmetic Slices II - Subsequence

The legal subsequence should have length >= 3. First we consider the length >= 2 situation. In this case, we can easily calculate the number of sequences by an O(n^2) dynamic programming. Let's denote the number of subsequences ended with A[i] and has a difference of d as dp[i][d]. dp[i][d] = sum(dp[j][d] if A[i] - A[j] == d). In fact, the number of sequence of length 2 is n * (n - 1) / 2. At last, we substract them from the answer.

964 - Least Operator to Express Number

We should consider the problem this way: Division operation can be only used to form 1. The expression would be something like this:

\[ T = \sum_{i=1}^{N} k_i x^i \]

\(k\) should be an integer and \(-x\lt k\lt x\). We can either append an \(x^i\) or an \(-x^i\) at the end of the expression. We have noticed that the cost for \(x^i\) is \(i\) for any \(i\gt 0\). For \(x^0\) the cost is 2. After summing up the cost, just substract the first operator out and that is the answer.

The calculation is not straightforward. However, we noticed that the problem can be down scaled. We can first decide the number of \(x^0\), and then consider placing how many \(x^1\) afterwards. Placing \(x^1\) is the same as placing \(x^0\) in \(\lfloor target/x\rfloor\) situation. So we can use a top down dp to decide to add \(target \% x\) or \(x - target \% x\) of them depending on the results of higher order terms.

Let's denote dp(i, target) as the best results achieved when putting \(x^i\) to \(target\). We can solve the transition equation: dp(i, target) = min(cost(i) * (target % x) + dp(i + 1, target / x), cost(i) * (x - target % x) + dp(i + 1, target / x + 1)).

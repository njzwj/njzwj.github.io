1568 - Minimum Number of Days to Disconnect Island

题面

给定一个n*m的二维0-1矩阵，每个四连通的1区域可以看作是一个岛屿。我们称图中只要有超过1个这样的岛屿的情况非连通。现在可以进行将一个位置1改写成0的操作。问最少需要多少次操作，能将岛屿变成非连通的。

题解

检查连通性，如果图中初始就有超过1个岛屿，那么只需要0步操作即可。

如果图中初始仅有1个岛屿，那么首先考虑一种仅进行1步操作的就可分割的情况。这种情况下被删除的1区块类似于两座岛屿的“桥”。这种情况可以用一种简单的方式检测——删除一个1，然后检查岛屿连通性。如果删除后连通性发生了变化，则只需进行1步操作即可。

超出两步操作的情形呢？可以证明，在任何类型的岛屿中，至少有1个“最多与另外2个方块紧密相连的”方块。如果0步或者1步无法分割，那么最多只需要删除2个方块即可。

检查连通性我们使用洪水填充即可。它拥有\(O(mn)\)的时间复杂度，算上枚举点，整体的复杂度是\(O((mn)^2)\)的。

1569 - Number of Ways to Reorder Array to Get Same BST

题面

给定一个\(0 \sim n-1\)的排列\(S\)，按顺序将元素插入到一个BST(二叉搜索树)中，得到树\(T\)。问有多少种与\(S\)不同的排列\(S'\)，使得依次将\(S'\)元素插入BST中，也能得到相同的树\(T\)。

题解

让我们回忆一下BST的建立过程——不断比较当前节点\(u\)的值\(u.value\)与要插入的值\(x\)，决定将\(x\)放入\(u\)的左子树\(u.l\)或者是右子树\(u.r\)中。如果对应的\(u.l\)或\(u.r\)不存在，那么在该处新建一个值为\(x\)的节点作为父节点的左/右子节点，否则不断递归查找。

不难发现，为了构建相同的树，对于所有的\(x, y\in S\)，如果在树中\(x\)是\(y\)的祖先，那么必须先插入\(x\)，再插入\(y\)。由此我们可以构建出一套偏序关系：\(x\)所代表子树上所有的节点都应该在\(x\)以后插入。到了这一步思路就很明确了：

由于树\(T\)是固定的，我们只用统计如何构造出这个树的方案数即可。所以定义\(dp[x]\)表示以x为根的子树有多少种构建方式。注意到如果\(x\)和\(y\)同属一个父节点\(u\)，那么\(dp[x]\)和\(dp[y]\)无关，并且不难得到\(dp[u]=C_{size(x)+size(y)}^{size(x)}dp[x]dp[y]\)。其中\(size(x)\)表示以\(x\)为根的子树大小，\(C\)为组合数。

程序具体的实现就是对构建出的树进行一次DFS，统计树大小，并且进行树上动态规划。边界条件显然是叶子节点\(dp[x]=1\)，转移方程如上一段所述。时间复杂度\(O(n)\)。

1575 - Count All Possible Routes

题面

在数轴上分布着位置不同的\(n\)个城市，他们中每一个分别在\(P[i]\)处。你驾车从\(S\)到\(T\)，并且初始状态有\(f\)的燃油，从\(i\)到\(j\)需要消耗\(|P[i]-P[j]|\)的燃油，问总共有多少种不同的走法。

题解

动态规划：\(dp[i][j]\)表示到\(i\)市，还剩\(j\)的燃油时，总共有多少种走法。不难得出转移方程： \(dp[i][j]=\sum_{k\in Cities}{dp[k][j + |P[i]-P[k]|]}\)。由于燃油是单向减少的，直接从\(j=f\)开始逆向遍历更新即可。时间复杂度为\(O(nf)\)。

1579 - Remove Max Number of Edges to Keep Graph Fully Traversable

题面

给定一个无向图\(G=(V, E)\)，我们将\(e\in E\)分为三种。第1种是只有Alice可以经过的，第2种是只有Bob可以经过的，第3种是两个人都可以经过的。问，最多删除多少条边，可以使得对于Alice和Bob来说，图依旧是连通的？

题解

先只考虑一般的无向图，显然，对于\(n\)个顶点来说，我们最少仅需要\(n-1\)条边即可实现连通，此时图为无根树。我们只需要运行一个生成树算法，即可得到一组所需要的边。

接下来考虑有Alice和Bob的情况，我们要优先保留类型3的边，因为类型3的边比类型1和2的边具有更强的连通效用，这样做可以让总边数边尽量多地被减少。我们依旧分别针对Alice和Bob在图上运行生成树算法，找到适用于Alice和Bob的两组边。不难发现，类型1和2的边是无关的，我们在删除的时候不必考虑它们之间的影响，只有类型3的边，最终必须要同时满足Alice和Bob的连通需求。

所以针对Alice，我们在类型1、3的边上运行生成树算法，并且优先插入类型3的边；针对Bob，我们在类型2、3的边上运行生成树算法，同样优先插入类型3的边。在运行Kruskal时，类型3边的结果是相同的，所以不用考虑两次运行结果不同的情况。时间复杂度是\(O(|E|+|V|)\)（不带排序的Kruskal、并查集）。

1585 - Check if String is Transformable with Substring Sort Operations

题面

给定两个仅包含数字（0到9）的串\(S\)和\(T\)，并且\(|S|=|T|\)。每次可以选择\(S\)的一个非空子串对它进行排序，使得这个子串单调非减，你可以连续对\(S\)进行任意多次这样的操作。问\(S\)是否可以被这样转换为\(T\)。

题解

乍看这题无从下手，\(|S| \leq 10^5\)的数据量显然无法用搜索解决。观察每次操作对\(S\)造成的影响，我们惊讶地发现：每次操作必然导致\(S\)的正序对增加或者不变。这个结论不难说明：因为每次排序操作的结果都是子串单调非减，对于跨子串的对来说，正序数量不受影响，在子串内，正序对数量一定是非减的。所以对整体来说，进行若干次操作涉及了正序对单调变化的过程。我们从正序对数量合法性入手，可以检查从\(S\)到\(T\)的合法变换是否存在。

正序对 对于所有\(0\leq i < j < |S|\), \(S[i] < S[j]\)成立的组的数量。定义和逆序对形式相同。记\(S\)的正序对数量为\(F(S)\)。

虽然从整体上我们得到了一个合法的约束，即\(F(S)\leq F(T)\)，但是这个约束过于松散，因为在这个约束下，可以有这样的情况：虽然其它数在变换后以之为起点的正序对数变多了，也会存在一些正序对变少的情况。我们可以具体地看下面的这个例子：

\[ \begin{aligned} S = 102 \\ T = 021 \end{aligned} \]

不难得出\(F(S) = 1 + 1 + 0 = 2\)，\(F(T)=2 + 0 + 0 = 2\)，但是显然对于1这个数来说，以它开头的正序对变少了，也是无法通过上述变换得到的。

我们采用一种更严格的判定：假设\(P1[i]\)表示所有的\(S[i] < S[j]\)，并且\(i < j\)的数量，相似地在\(T\)上定义\(P2[i]\)。我们统计这两个正序对数的直方图。假设一个数变换前它位于\(S[u]\)，变换后位于\(T[v]\)，我们要求\(P1[u] <= P2[v]\)（约束1）。这是一个比整体正序对数更加严格的正序对约束。

为了方便下面的说明，我们定义\(P\)的直方图\(Q\)，\(Q[x]\)表示\(P[i]=x\)的\(i\)的数量，相似在\(P1\)和\(P2\)上定义\(Q1\)和\(Q2\)。我们发现，约束1等价于\(\sum_{i=0}^{i\leq x} Q1[i] \leq \sum_{i=0}^{i\leq x} Q2[i]\)全部成立。我们可以用程序轻易求出\(Q\)，从而判断结果。

由于字符串每个位置只有10种值，求解逆序对是\(O(|S|)\)的，总的时间复杂度是\(O(|S|)\)。
